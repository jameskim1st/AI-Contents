실제로 동작하지는 않지만, 완성된 것처럼 보이는 디자인 시안
코드를 한 줄도 쓰기 전에 "완성되면 이렇게 생길 거야"를 미리 보여주는 화면 시안입니다. 요즘은 AI에게 "이런 느낌의 대시보드 목업 만들어줘"라고 하면 바로 만들어줍니다.

왜 중요한가요? 코드 작성에 시간을 투자하기 전에, "이런 방향이 맞나요?"를 빠르게 합의할 수 있어요.
모델하우스와 같아요. 실제 아파트를 짓기 전에 "완성되면 이런 느낌"을 보여주는 샘플 공간.

MVP (Minimum Viable Product)
최소 기능 제품 — 핵심 기능만 넣어서 빠르게 만든 첫 번째 버전
처음부터 완벽한 서비스를 만드는 게 아니라, 가장 핵심적인 기능만 넣어서 빠르게 출시하고, 사용자 반응을 보면서 개선해나가는 전략입니다.

바이브코딩이 특히 MVP에 강력한 이유: AI에게 핵심 기능만 지시하면 몇 시간 만에 동작하는 프로토타입을 만들 수 있어요.

MVP의 핵심 사고방식: "이 서비스에서 단 하나의 기능만 남긴다면?" → 그게 MVP입니다.
200페이지짜리 완벽한 보고서를 3개월 만드는 대신, 핵심 결론 5페이지를 먼저 보여주고 피드백을 받는 것과 같습니다.

AI가 만들어준 결과물, 이게 다 뭐지?

AI에게 "웹사이트 만들어줘"라고 했더니 코드가 쏟아져 나왔어요. 파일 이름 끝에 .html, .css, .js가 붙어 있는데... 이걸 이해하려면, 웹페이지가 어떻게 만들어지는지를 직접 눈으로 보는 게 가장 빠릅니다.

🧪 직접 비교하세요 — HTML만 있을 때 vs CSS·JS가 입혀졌을 때

웹은 어떻게 작동하는가?

AI가 만들어준 코드가 실제로 사용자에게 어떻게 전달되는지 — 인터넷의 기본 구조를 알면 바이브코딩이 훨씬 수월해집니다.

먼저 인터넷에서 누가 요청하고 누가 응답하는지 — 기본 등장인물부터 알아봅시다.

AI가 만든 코드, 어디서 돌려볼까?

AI가 코드를 만들어줘도, 그 코드를 실행하고 결과를 확인할 공간이 필요합니다. AI가 PPT 초안을 만들어줬다고 해도, 결국 파워포인트를 열어서 확인하고 다듬잖아요? 코드도 마찬가지입니다. 이때 사용하는 전용 도구가 바로 IDE예요.

IDE (Integrated Development Environment)
통합 개발 환경 — 코드를 쓰고, 실행하고, 관리하는 올인원 도구
❓ 왜 필요한가요?
AI가 코드를 만들어줘도 그 코드를 어딘가에 붙여넣고, 실행하고, 결과를 확인해야 합니다. 메모장에서도 가능하지만, 메모장은 코드의 오타를 알려주지 않고, 실행 버튼도 없고, 파일 관리도 불편합니다.

IDE는 이 모든 걸 한 곳에서 해결해주는 도구예요. 코드에 자동으로 색을 입혀서 가독성을 높여주고(문법 강조), 오타나 실수에 빨간 밑줄을 쳐주고(에러 감지), 프로젝트의 모든 파일을 깔끔하게 관리해줍니다.

❓ 왜 VS Code인가요?
무료이고, 전 세계 개발자의 약 70%가 사용할 만큼 생태계가 크며, 확장 기능이 풍부합니다. 그리고 무엇보다, AI 기반 코딩 도구(Cursor, Copilot 등)가 거의 다 VS Code를 기반으로 만들어져 있어서, 이걸 익혀두면 다른 도구로 넘어가기도 쉬워요.
PPT를 만들 때 파워포인트를 쓰는 이유와 같습니다. 메모장에 글과 도형을 그릴 수는 없잖아요? 코드도 마찬가지로, 전용 도구가 있어야 편하게 작업할 수 있습니다. IDE는 코드 세계의 파워포인트예요.

❓ 이게 뭔가요?
우리가 컴퓨터를 사용할 때 보통 마우스로 폴더를 클릭하고, 아이콘을 더블클릭해서 프로그램을 실행하죠? 이게 GUI(그래픽 사용자 인터페이스)입니다. 터미널은 이와 반대로, 텍스트 명령어를 직접 타이핑해서 컴퓨터에게 지시하는 방식이에요. IDE 하단에 보이는 까만 화면이 바로 터미널입니다.

❓ 비개발자인 저도 이걸 써야 하나요?
네, 하지만 걱정하지 마세요. 바이브코딩에서 터미널을 쓰는 이유는 딱 두 가지입니다:

① 패키지 설치 — AI가 만든 코드에 필요한 부품(패키지)을 설치할 때 npm install 같은 명령어를 터미널에 입력합니다. GUI 버튼이 따로 없어서 터미널이 필요해요.
② 코드 실행 — "이 코드를 돌려줘"라는 명령(npm start 등)도 터미널에 입력합니다.

핵심은, 이 명령어를 외울 필요가 없다는 점입니다. AI가 "터미널에 이걸 입력하세요"라고 알려주면, 그대로 복사해서 붙여넣기만 하면 됩니다.
카카오톡에서 메시지를 보내듯이, 터미널에서는 컴퓨터에게 명령어를 보냅니다. "이 폴더 열어줘", "이 프로그램 실행해줘" 같은 명령을 텍스트로 보내는 채팅창이라고 생각하세요.

🔑 도구보다 중요한 것

어떤 도구를 쓰든 핵심 원리는 같습니다: AI에게 명확히 지시하고, 결과를 확인하고, 피드백을 반복하는 것. 도구는 언제든 바꿀 수 있으니, 하나를 골라서 직접 만들어보는 경험이 가장 중요해요.

"ChatGPT나 Gemini에서도 코드가 나오는데, 위의 바이브코딩 도구들은 뭐가 다른 걸까?" — 도구들을 봤으니 이제 그 차이가 더 선명하게 보일 거예요.

💬 AI 앱

ChatGPT · Gemini · Claude (채팅)
AI가 코드를 텍스트로 보여줍니다.
코드를 받아서 내가 직접 파일을 만들고, 환경을 세팅하고, 실행해야 해요.
1.AI가 코드를 채팅창에 출력 2.내가 코드를 복사 3.IDE를 열어서 붙여넣기 4.패키지 직접 설치 (npm install...) 5.직접 실행해서 결과 확인 6.안 되면 에러 메시지를 다시 복사해서 AI에게...
😓 코드 받기 → 환경 세팅 → 실행까지 전부 내 일

🛠️ 바이브코딩 도구

Bolt.new · Lovable · Cursor · Claude Code · Replit
AI가 코드를 생성하고, 실행하고, 미리보기까지 한 번에 처리합니다.
내가 할 일은 자연어로 지시하고 결과를 확인하는 것뿐이에요.
1.프롬프트 입력 2.AI가 코드 생성 + 패키지 설치 + 실행 자동 3.미리보기 화면이 바로 옆에 표시 4."이 부분 수정해줘" → 즉시 반영 5.완성되면 Deploy 버튼 한 번 6.URL 생성 → 공유 끝!

지금까지 배운 개념들이 실제로 어떻게 연결되는지 살펴봅시다. 워크플로우 탭에서 전체 흐름을, 환경 세팅 가이드 탭에서 실습 전 준비 사항을 확인하세요.

아이디어 하나를 서비스로 만드는 전체 흐름입니다. 이 워크플로우를 머릿속에 그릴 수 있으면, 바이브코딩을 시작할 준비가 된 겁니다.

🗺️ 아이디어에서 서비스까지 — 각 단계를 클릭해서 자세히 보세요
💡
STEP 1아이디어 구체화
"어떤 문제를 해결할 것인가?" — MVP 사고법으로 핵심 기능 정의
▾
모든 프로젝트는 "어떤 문제를 해결하고 싶은가?"에서 시작합니다. 이 단계에서 Ch.05의 MVP 사고법을 적용해, 핵심 기능만 정의하세요.
❌ 이렇게 시작하면 안 됨
"SNS도 되고, 쇼핑몰도 되고, AI 추천도 되는 앱을 만들자"
✅ 이렇게 시작하면 좋음
"팀 프로젝트 진행률을 한눈에 볼 수 있는 대시보드를 만들자"
🎨
STEP 2도구 선택 & 목업 요청= 작업 흐름 1단계
바이브코딩 도구를 열고, 템플릿으로 목업을 요청
▾
Ch.05에서 배운 도구 중 하나를 골라서, 앞서 배운 프롬프트 템플릿을 붙여넣고 빈칸을 채운 뒤 AI에게 보내세요. 맨 아래 "우선 목업으로 보여줘" 문장이 핵심입니다.
"프로젝트 관리 대시보드를 만들어줘.

기능 요구사항:
- 프로젝트 목록이 카드 형태로 보임
- 각 카드에 진행률 바가 있음
- 상태별(진행중/완료) 필터 기능

디자인:
- 다크 모드, 모바일 반응형

우선 목업으로 보여줘."
📌 팀원이나 상사에게 이 목업을 보여주고 "이 방향 맞나요?"를 확인하세요.
💬
STEP 3디자인 피드백= 작업 흐름 2단계
목업을 보면서 2~3회 수정 요청 → 시각적 완성도 80% 이상 잡기
▾
목업을 보면서 한 번에 하나씩 수정을 요청합니다. "전체적으로 바꿔줘"보다 구체적인 지시가 훨씬 정확해요.

✅ 반영

✅ 확정
💡 시각적 완성도를 80% 이상 잡는 게 목표입니다. 기능은 다음 단계에서!
⚙️
STEP 4기능 입히기 → MVP 완성!= 작업 흐름 3단계
"디자인 확정이야. 기능을 넣어줘" → 한 번에 하나씩 추가
▾
디자인 확정 후 "디자인 확정이야. 이 화면 그대로 유지하면서 실제 기능을 넣어줘"라고 보냅니다. 기능도 한 번에 하나씩 추가하세요.

Git 설치

GitHub 계정 생성

AI 서비스 API 키 발급 (선택)
