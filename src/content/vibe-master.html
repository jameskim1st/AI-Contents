Chapter 02

왜 Claude Code인가

AI 코딩 도구는 크게 3단계로 진화해 왔습니다. 자동완성 → 대화형 채팅 → 에이전틱 코딩. Claude Code는 3단계에 해당합니다.

AI 코딩 도구의 진화 3단계
1단계
자동완성
IDE가 다음 줄을 예측
반복 패턴에 탁월
한계: 현재 파일로 컨텍스트 제한
2단계
대화형 채팅
코드 토론, 버그 설명
스니펫 붙여넣기
한계: 다중 파일 조율이 수동
3단계
에이전틱 코딩
목표를 분해하고 독립 실행

Chapter 03

AI 시대의 개발 방법론 — TDD vs SDD

에이전틱 코딩이 등장하면서 병목이 이동했습니다. 이전의 병목은 "코드를 작성하는 것", 지금의 병목은 "AI에게 무엇을 시킬지 정하는 것 + 결과를 확인하는 것"입니다.

Chapter 04

하네스 & 컨텍스트 엔지니어링

Agent = Model + Harness. 모델이 아닌 것은 전부 하네스입니다. 하네스가 결정하는 것 — 무엇을 보는지(컨텍스트) · 무엇을 할 수 있는지(도구·권한) · 언제 멈추는지(제약) · 잘못됐을 때(복구)

조합하는 패턴 — Plan-Critic-Build, Ralph Loop

Chapter 06

이렇게 하면 망한다 — 한계와 실패 패턴

도구를 잘 쓰는 것만큼 중요한 건, 잘못 쓰는 패턴을 아는 것입니다. 95% 성공률의 에이전트가 20단계 체인을 거치면 전체 성공률은 36%로 떨어집니다.
