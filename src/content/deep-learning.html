01

뉴런과 퍼셉트론 — 딥러닝의 최소 단위

딥러닝은 결국 인공 뉴런이 수백만 개 모인 것입니다. 모든 것은 1957년의 단 하나의 뉴런, 퍼셉트론에서 시작했습니다. 생물학적 뉴런의 작동 방식을 기계로 옮기면 무엇이 되는지 보겠습니다.

🔬 생물학적 뉴런 → 인공 뉴런
🧬 생물학적 뉴런
자극(전기 신호)이 수상돌기로 들어와 → 세포체에서 합산 → 임계값을 넘으면 축삭으로 발화(fire).
🤖 인공 뉴런 (Perceptron)
입력(x) × 가중치(w) → 합산(Σ) + 편향(b) → 활성화 함수(σ) → 출력(y). 생물학적 뉴런과 1:1 대응!
📜 퍼셉트론의 탄생 — 1957
🧑‍🔬
Frank Rosenblatt (코넬대 심리학자, 신경과학 연구원, 최초의 실용적 인공 뉴런 발명자)가 1957년 퍼셉트론을 발명합니다. 뇌의 뉴런을 수학 모델로 옮긴 최초의 실용적 시도. 뉴욕타임즈는 "기계가 걷고, 말하고, 쓰고, 스스로 복제하고, 자기 존재를 인식하게 될 것"이라고 보도했습니다. 하지만 이 열광은 12년 후 차갑게 식습니다.
🔌 퍼셉트론으로 논리 게이트 만들기
AND 게이트
w₁=1, w₂=1, b=-1.5 → 둘 다 1일 때만 출력 1
OR 게이트
w₁=1, w₂=1, b=-0.5 → 하나만 1이어도 출력 1
❌ XOR 문제 — 단층 퍼셉트론의 한계
❌ 단층 퍼셉트론 — 실패
XOR: 둘이 다르면 1, 같으면 0. 직선 하나로는 절대 분리할 수 없습니다. 1969년 Marvin Minsky (MIT AI 연구소 공동창립자, "인공지능"이란 용어를 대중화한 선구자)가 이를 수학적으로 증명하며 1차 AI 겨울이 시작됩니다.
✅ 다층 퍼셉트론(MLP) — 해결!
은닉층(Hidden Layer)을 하나 추가하면 직선 2개를 결합할 수 있고, XOR를 해결합니다. 이것이 다층 퍼셉트론(MLP) — 딥러닝의 출발점입니다.
🎯 핵심: 퍼셉트론 하나는 직선 하나를 그을 수 있습니다. 층을 쌓으면(Deep) 곡선, 곡면, 초고차원 경계까지 표현할 수 있습니다. 이것이 "딥(Deep)" 러닝의 의미입니다.
02

활성화 함수 비주얼 가이드 — 뉴런의 ON/OFF 스위치

뉴런이 받은 신호를 그대로 전달하면 안 됩니다. 선형 연산만 쌓으면 아무리 깊어도 결국 직선 하나와 같습니다. 활성화 함수는 각 뉴런의 "반응 임계값" — 자극이 충분하면 발화하고, 아니면 무시합니다.

🤔 왜 비선형이 필요한가?
😐 선형만 쌓으면?
f(x) = 2x + 1, g(x) = 3x - 2
f(g(x)) = 2(3x-2)+1 = 6x - 3 ... 여전히 직선!
🎯 비선형을 넣으면!
비선형 활성화를 사이에 넣으면 곡선, 곡면, 어떤 복잡한 패턴이든 근사(approximate)할 수 있습니다. 이것이 Universal Approximation Theorem.
📊 4대 활성화 함수 — 함수 곡선 + 미분 곡선
① Sigmoid
σ(x) = 1 / (1 + e⁻ˣ)  →  출력: 0~1
✅ 확률 해석 가능
❌ 기울기 소실!
② Tanh
tanh(x) = (eˣ - e⁻ˣ)/(eˣ + e⁻ˣ)  →  출력: -1~1
✅ 영점 대칭
❌ 여전히 기울기 소실
③ ReLU ⭐ 가장 많이 사용
ReLU(x) = max(0, x)  →  초간단!
✅ 초고속 · 기울기 보존
❌ 죽은 뉴런 (x<0 → 영원히 0)
④ GELU — Transformer 표준
GELU(x) = x · Φ(x)  →  매끄러운 ReLU
✅ 매끄러움 · 죽은 뉴런 없음
GPT, BERT, Claude 전부 사용
💡 비유: 활성화 함수는 뉴런의 "반응 임계값"입니다. 교실에서 손드는 것과 같아요 — 확신이 일정 수준을 넘으면 손을 들고(발화), 아니면 가만히 있습니다(비활성). Sigmoid는 "소심한 학생"(항상 조심스럽게), ReLU는 "직설적 학생"(0 아니면 전부), GELU는 "섬세한 학생"(작은 자극도 약간은 반영).
03

Forward Propagation — 입력에서 출력까지

Forward Pass는 공장 조립 라인입니다. 원재료(입력 데이터)가 단계별로 가공되어 완제품(예측값)이 나옵니다. 이 과정을 숫자로 하나하나 따라가 봅시다.

🤔 Forward Pass가 뭔데 중요한가요?
신경망의 학습은 딱 두 단계입니다:
① Forward Pass: 입력 데이터를 넣고 → 결과를 예측 → "얼마나 틀렸는지" 확인
② Backward Pass (역전파): 틀린 만큼 → 가중치를 조정

비유: 시험 보기(Forward) → 채점 → 오답노트 정리(Backward)
이 두 단계를 수천~수만 번 반복하면 모델이 점점 정확해집니다.

이번 챕터는 ①번, "시험 보는 과정"입니다. 다음 챕터(Ch.04)에서 "오답노트 정리"를 배웁니다.
🏭 비유: 공장 조립 라인 = Forward Pass
🔢 구체 숫자로 따라가기 — 2층 네트워크 Forward Pass
Step 1: 입력
x = [1.0, 0.5]
Step 2: 가중치 곱셈
w = [0.6, -0.3]
b = 0.1
z = 0.6×1.0 + (-0.3)×0.5 + 0.1
z = 0.55
Step 3: 활성화 (ReLU)
h = ReLU(0.55)
= max(0, 0.55)
h = 0.55
Step 4: 출력층
w₂ = 0.8, b₂ = -0.1
z₂ = 0.8×0.55 + (-0.1)
z₂ = 0.34
Step 5: 최종 출력
ŷ = σ(0.34)
= 1/(1+e⁻⁰·³⁴)
ŷ = 0.584
Step 6: 손실 계산
정답 y = 1.0
Loss = (1.0 - 0.584)²
Loss = 0.173
💡 Forward Pass는 "질문을 던지는 것"입니다. 네트워크가 "이 입력의 답은 0.584일 것이다"라고 예측합니다. 정답은 1.0이므로 틀렸습니다. 이 오차(Loss = 0.173)를 줄이려면? 다음 챕터의 역전파(Backpropagation)로 가중치를 교정합니다.
➡️
Forward Pass
시험지를 풀고 답을 제출하는 것
⬅️
Backpropagation
채점 후 오답노트를 작성하는 것
04

역전파 (Backpropagation) — 딥러닝 학습의 심장 ⭐

딥러닝에서 가장 중요한 알고리즘입니다. Forward Pass로 예측한 결과가 틀렸을 때, 오차를 출력에서 입력 방향으로 역추적하며 각 가중치가 오차에 얼마나 기여했는지 계산합니다. 비유하면 "시험 채점 후 오답노트" — 어디서, 얼마나 틀렸는지 역추적하는 과정입니다.

🎯 "미분"이 처음이라면?
미분은 어려운 수학이 아닙니다. 한마디로 "변화의 정도"입니다.

예를 들어, 자동차 속도 = "위치의 변화 정도". 시속 60km는 "1시간에 위치가 60km 변한다"는 뜻이죠. 마찬가지로 신경망에서 미분은 "가중치를 조금 바꾸면 결과가 얼마나 바뀌는가?"를 측정합니다.

Chain Rule = 도미노 효과 계산법
도미노 A가 B를 쓰러뜨리고, B가 C를 쓰러뜨린다면?
"A가 C에 미치는 영향" = "A→B 영향" × "B→C 영향"
이것이 Chain Rule의 전부입니다. 곱하기만 하면 됩니다!
🔗 Chain Rule — 역전파의 수학적 핵심
📐 합성함수의 미분
함수가 체인처럼 연결되어 있으면:
f(g(x))의 미분 = f'(g(x)) × g'(x)
예시: f(x) = x², g(x) = 3x + 1
f(g(x)) = (3x+1)²
미분 = 2(3x+1) × 3 = 6(3x+1)
각 단계의 미분을 곱해나가면 됩니다!
🎯 비유: 도미노 효과
각 도미노가 다음 도미노에 미치는 영향을 역으로 추적합니다. 마지막 도미노(출력 오차)에서 시작해서 첫 도미노(입력층 가중치)까지.
🔄 3층 네트워크 역전파 — 단계별 시각 투어
📖 기호 읽는 법
∂L/∂w₃ 이런 기호가 무섭게 보이시죠? 사실 아주 간단합니다:
"∂L/∂w₃" = "가중치 w₃를 살짝 바꾸면 손실(L)이 얼마나 변하나?"

비유: 라면 끓일 때 물 양(w₃)을 살짝 바꾸면 맛(L)이 얼마나 달라지나?
이 "얼마나 달라지나"가 바로 ∂L/∂w₃입니다. 숫자가 크면 "물 양이 맛에 큰 영향", 작으면 "별 영향 없음".
📋 역전파 7단계 — 하나씩 따라가기
Step 1
손실(L) 계산
Forward Pass 결과 ŷ=0.58, 정답 y=1.0
L = (1.0 - 0.58)² = 0.176
Step 2
∂L/∂ŷ 계산
출력이 오차에 미친 영향
= 2(ŷ - y) = -0.84
Step 3
∂L/∂w₃ 계산
출력층 가중치의 영향
= ∂L/∂ŷ × ∂ŷ/∂w₃
Step 4
은닉층2로 전파
∂L/∂h₂ = ∂L/∂ŷ × ∂ŷ/∂h₂
Chain Rule로 이어짐!
Step 5
∂L/∂w₂ 계산
은닉층 가중치의 영향
gradient가 점점 작아짐
Step 6
은닉층1로 전파
∂L/∂h₁ — 오차가 더 깊이 전달
기울기가 급격히 약해질 수 있음
Step 7
∂L/∂w₁ 계산
입력층 가중치까지 도달!
이 값으로 w₁을 업데이트
😱 기울기 소실 문제 (Vanishing Gradient)
Sigmoid 활성화 함수의 미분 최댓값은 0.25입니다. 층을 지날 때마다 gradient에 0.25를 곱하면?
결과: 5층만 지나도 gradient가 0.004 — 거의 0! 초기 층의 가중치는 사실상 학습을 못합니다. 이것이 1990~2000년대 딥러닝이 불가능했던 핵심 이유입니다.
🎤 마이크 볼륨 비유
Sigmoid의 미분 최댓값이 0.25라는 건 이런 뜻입니다:

마이크 → 앰프1(볼륨 25%) → 앰프2(볼륨 25%) → 앰프3(볼륨 25%) → 스피커

원래 소리를 100이라 하면:
앰프1 후: 100 × 0.25 = 25
앰프2 후: 25 × 0.25 = 6.25
앰프3 후: 6.25 × 0.25 = 1.56
5개 층 후: 0.098 — 거의 안 들림!

이것이 기울기 소실입니다. 신호(기울기)가 층을 지날수록 사라져서, 앞쪽 층은 "뭘 고쳐야 하는지" 정보를 못 받습니다.

ReLU의 해결: ReLU의 미분은 양수 영역에서 1입니다. 볼륨 100%! 신호가 줄지 않고 그대로 전달됩니다.
⚡

x>0이면 미분=1
기울기가 안 사라짐!

🔁 실제 학습 루프 — Forward → Loss → Backward → Update → 반복
🎯 역전파의 본질: "내가 틀린 이유를 출력에서 입력까지 역추적하고, 각 가중치에 '너는 오차의 몇 %를 책임진다'고 알려주는 것." 이 알고리즘은 1986년 David Rumelhart, Geoffrey Hinton, Ronald Williams (Rumelhart: 심리학자·인지과학자, Hinton: 토론토대 교수·2018 튜링상·"딥러닝의 대부", Williams: 미 노스이스턴대 교수)가 발표한 논문으로 딥러닝의 2차 봄을 열었습니다.
🎯 역전파 한 줄 요약 — 이것만 기억하세요!
Forward Pass: 예측 → 틀렸다!
Backward Pass: "어디서 얼마나 틀렸는지" 역추적
가중치 업데이트: 틀린 만큼 조정
반복: 수천 번 → 점점 정확해짐
이것이 모든 딥러닝 학습의 핵심입니다.
GPT도, CNN도, LSTM도 — 전부 이 과정으로 학습합니다.
수학이 어려워도 이 흐름만 이해하면 충분합니다!

→ 느리고, 평평한 곳에서 멈출 수 있음

Momentum (스키 타는 사람): "관성이 있어서 평평한 곳도 미끄러져 지나가요."
→ 빠르지만, 너무 빨라서 최적점을 지나칠 수 있음

→ 가장 똑똑하게 최적점을 찾음

결론: 2026년 현재 대부분의 딥러닝 모델은 Adam을 사용합니다. 여러분이 직접 모델을 만들 일이 없더라도, "Adam이 가장 널리 쓰이는 옵티마이저"라는 것만 기억하세요.

CNN — 이미지를 이해하는 신경망

Convolutional Neural Network는 인간의 시각 피질을 모방합니다. "돋보기로 이미지를 훑으며 패턴을 찾는" 방식 — 이것이 딥러닝이 이미지를 이해하는 방법입니다.

💡 필터가 왜 이미지 위를 "슬라이딩"하나요?
고양이 사진에서 "귀"를 찾는다고 합시다.
귀가 사진 왼쪽에 있을 수도, 오른쪽에 있을 수도 있잖아요?
그래서 필터가 이미지 전체를 훑으면서 "여기 귀 패턴이 있나? 없나?" 확인합니다.

이것이 CNN의 가장 큰 장점입니다: 패턴이 어디에 있든 찾을 수 있다! (이를 위치 불변성이라 합니다)
🔍 Convolution 연산 — 필터가 이미지를 훑는다
🔍 "왜 이렇게 곱하고 더하나요?"

필터(커널)는 "찾고 싶은 패턴"입니다.
예를 들어, 세로 선(|)을 찾는 3×3 필터는 가운데 열만 1이고 나머지는 0입니다:

0 1 0
0 1 0
0 1 0

이 필터를 이미지 위에 놓고 곱하면?
• 세로 선이 있는 곳 → 값이 크게 나옴 (패턴 발견!)
• 세로 선이 없는 곳 → 값이 작게 나옴 (패턴 없음)

즉, Convolution = "이미지에서 특정 패턴이 어디 있는지 찾기"
CNN은 이런 필터를 수십~수백 개 사용해서 선, 곡선, 텍스처, 눈, 코, 입... 점점 복잡한 패턴을 찾아냅니다. 그리고 이 필터의 값(숫자)은 사람이 정하는 게 아니라 학습으로 자동 결정됩니다!

RNN · LSTM — 시퀀스를 이해하는 신경망

시간 순서가 중요한 데이터 — 텍스트, 음성, 주가, 날씨. CNN이 공간의 패턴을 잡는다면, RNN은 시간의 패턴을 잡습니다. 소설을 읽을 때 이전 내용을 기억하며 다음 페이지를 읽는 것처럼.

🔁 RNN 기본 구조 — 기억하는 뉴런
Folded View (접힌 모습)
Unrolled View (시간축으로 펼침)
🔁 "같은 가중치를 재사용한다"는 무슨 뜻?

소설을 읽을 때 같은 뇌로 1페이지도, 100페이지도, 500페이지도 읽잖아요?
페이지마다 다른 뇌를 쓰지 않죠.

RNN도 마찬가지입니다. 시점 t=1이든 t=100이든 같은 가중치(같은 뇌)로 처리합니다.
덕분에: ① 모델 크기가 작고 ② 시퀀스 길이에 상관없이 처리 가능합니다.

단점: 같은 뇌로 너무 긴 소설을 읽으면 앞부분을 잊어버리죠. 이것이 Vanishing Gradient 문제이고, LSTM이 해결한 겁니다.
📞 RNN의 한계 — 전화 돌리기 비유
❌ RNN의 문제: 긴 기억 실패
비유: 전화 돌리기. 20명을 거치면 원래 메시지가 왜곡됩니다. RNN에서 50 타임스텝 전의 정보는 gradient가 소실되어 사실상 학습 불가.
✅ LSTM: 스마트 수첩
LSTM (Long Short-Term Memory)은 Sepp Hochreiter (독일 뮌헨공대 출신, 현 요하네스 케플러 대학 교수, 장기 의존성 문제의 핵심 해결자)와 Jurgen Schmidhuber (스위스 IDSIA 연구소 공동소장, 재귀 신경망 연구의 선구자)가 1997년 발표. 3개의 게이트로 정보를 관리합니다:
🚪 Forget Gate: 이전 기억 중 버릴 것 결정 (DELETE)
📝 Input Gate: 새로운 정보 중 기억할 것 결정 (WRITE)
📤 Output Gate: 기억에서 출력할 것 결정 (READ)
📱 LSTM = 스마트 노트 앱

LSTM의 3개 게이트를 노트 앱으로 이해해보세요:

① Forget Gate (삭제 버튼)
"어제 날씨 정보는 더 이상 필요 없으니 삭제" → 0에 가까운 값 = 삭제, 1에 가까운 값 = 유지
실제 동작: 기존 기억 × 0.1 = 거의 삭제 / 기존 기억 × 0.9 = 거의 유지

② Input Gate (쓰기 버튼)
"오늘 새로 배운 단어를 추가 노트에 기록" → 중요도에 따라 얼마나 강하게 기록할지 결정
실제 동작: 새 정보 × 0.8 = 강하게 기록 / 새 정보 × 0.2 = 약하게 기록

③ Output Gate (출력 버튼)
"지금 이 질문에 답하려면 노트에서 어떤 부분을 꺼내야 하지?" → 현재 맥락에 맞는 정보만 출력

핵심: 이 세 게이트 덕분에 LSTM은 "뭘 기억하고, 뭘 잊고, 뭘 출력할지"를 스스로 학습합니다. 단순 RNN은 이 능력이 없어서 긴 문장에서 앞부분을 잊어버렸죠.
🧠 LSTM 셀 내부 구조 — 컨베이어 벨트 + 3 게이트
💡 비유: LSTM = 스마트 수첩. 어떤 정보를 기억(Keep)할지, 새로 쓸지(Write), 출력할지(Read)를 매 시점마다 결정합니다. 핵심은 셀 상태(Cell State)라는 "컨베이어 벨트" — 정보가 거의 변형 없이 먼 과거에서 미래로 전달됩니다.
🚀 그런데 2017년, 이 모든 걸 뒤집은 논문이 나옵니다. "Attention Is All You Need" — RNN 없이, 순환 연결 없이, 모든 위치를 동시에 보는 Transformer가 등장합니다. LSTM의 순차 처리 한계를 Self-Attention의 병렬 처리로 완전히 대체. 이 이야기는 Ch.10에서 이어집니다.
08

Overfitting 대응 — 학습의 함정을 피하는 법

모델이 학습 데이터를 너무 잘 외우면 새로운 데이터에서 실패합니다. 비유하면 "시험 답을 통째로 외운 학생" — 같은 문제는 만점이지만 새 문제는 0점입니다. 이것이 Overfitting입니다.

📉 Overfitting을 한 눈에 — Train vs Test 정확도
🛡️ Overfitting 대응 5가지 전략
① Dropout
학습 때 랜덤으로 뉴런을 끄기 (보통 20~50%).
비유: 팀원이 매번 달라지는 회의 — 특정 사람에 의존하지 않는 조직이 됨.
② Batch Normalization
[-5, 12, -3, 8]
→
[-.5, 1.2, -.3, .8]
각 층의 출력을 평균 0, 분산 1로 정규화. 학습을 안정화하고 더 높은 Learning Rate 사용 가능.
③ Early Stopping
Validation Loss가 올라가기 시작하면 학습을 즉시 중단. 가장 간단하고 효과적.
④ Data Augmentation
🖼️ 원본
🔄 회전
↔️ 반전
🔍 확대
이미지를 회전, 뒤집기, 확대, 색상 변환으로 데이터를 뻥튀기. 1000장 → 10000장 효과.
⑤ Regularization (L1/L2)
Loss = Loss_원래 + λ × Σ|w|²
가중치가 너무 커지지 않게 페널티를 부여. 모델이 단순한 패턴을 선호하게 유도.
📊 데이터 3분할 — Train / Validation / Test
💡 핵심 원칙: Test 데이터는 마지막 한 번만 사용합니다. 학습 중에 Test로 확인하면 그것도 Overfitting! Validation 데이터가 학습 중 성능 체크 역할을 맡습니다.
