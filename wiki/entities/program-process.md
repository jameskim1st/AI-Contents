# Program & Process (프로그램과 프로세스)

**Category:** IT 기초 / 소프트웨어
**Status:** stable
**Last updated:** 2026-04-12

## TL;DR

개발자가 짠 **코드**를 컴파일하면 **프로그램**이 되고, 프로그램을 실행하면 메모리 위에 올라간 **프로세스**가 된다. 프로그래밍 언어는 고급언어(Python, Java, JS)와 저급언어(기계어)로 나뉘며, 컴파일러가 이 변환을 담당한다. 빌드는 소스코드를 실행 가능한 산출물로 만드는 전체 과정이고, IDE는 코딩부터 디버깅·컴파일까지 한곳에서 할 수 있는 통합 개발 환경이다.

## 설명

### 코드 → 프로그램 → 프로세스 흐름

```
소스 코드 (Python, Java 등)
    ↓  컴파일 / 빌드
프로그램 (실행 가능한 파일)
    ↓  실행 (더블클릭, 명령어)
프로세스 (메모리 위에서 동작 중)
    ↓  종료
메모리에서 제거
```

- **프로그램** = 코드(명령어)의 집합이 컴퓨터가 이해할 수 있는 형태로 컴파일된 것
- **프로세스** = 실행 중인 프로그램. 카톡, 브라우저, Excel을 실행하면 각각 프로세스가 된다
- 작업관리자(Task Manager)에서 프로세스 목록을 확인하고 강제 종료할 수 있다

### 프로세스의 메모리 구조

프로세스가 생성되면 메모리 안에 다음 영역이 할당된다:

| 영역 | 역할 | 예시 |
|---|---|---|
| **Stack** | 함수 호출 정보, 지역 변수 저장 | 함수 A → B → C 호출 순서 기록 |
| **Heap** | 동적으로 할당되는 데이터 | 사용자 입력, 파일 읽기 결과 등 |
| **Code** | 프로그램의 명령어 자체 | 컴파일된 기계어 코드 |
| **Data** | 전역 변수, 상수 | 프로그램 시작 시 초기화되는 값 |

### 인터럽트 (Interrupt)

여러 프로그램이 동시에 실행 중일 때, 키보드 입력이나 마우스 클릭 같은 **갑작스러운 신호**가 발생하면 OS가 현재 CPU 작업을 잠시 멈추고 해당 신호를 먼저 처리한다. 이것이 **인터럽트**다. 처리 속도가 매우 빨라서 사용자는 프로그램이 멈추는 느낌을 받지 않는다.

### 프로그래밍 언어: 고급 vs 저급

- **고급언어** — 사람이 읽기 쉬운 언어 (Python, Java, JavaScript, C++ 등)
- **저급언어** — 컴퓨터(CPU)가 직접 이해하는 언어 (기계어, 어셈블리어)
- CPU는 0과 1(기계어)만 읽을 수 있으므로, 고급언어 → 저급언어로 변환이 필요

### 컴파일러와 바이트코드

- **컴파일러(Compiler)** — 고급언어를 저급언어(기계어)로 변환하는 시스템 프로그램
- **바이트코드(Bytecode)** — 컴파일 결과가 기계어가 아니라 **소프트웨어 가상 머신**이 해석하는 중간 코드
  - Java → JVM(Java Virtual Machine)이 바이트코드 해석
  - Python → CPython 인터프리터가 바이트코드 해석
  - 장점: "한 번 컴파일하면 어디서든 실행" (Write Once, Run Anywhere)

### 빌드 (Build)

**소스코드를 실행 가능한 소프트웨어 산출물로 만드는 전체 과정.** 컴파일은 빌드 과정의 일부다. 우리가 다운받아 클릭으로 바로 실행하는 프로그램은 모두 빌드를 거친 결과물이다. 바이브코딩에서 `npm run build` 명령어가 바로 이 빌드 과정이다.

### IDE (Integrated Development Environment)

코딩, 디버깅, 컴파일, 실행을 **하나의 프로그램 안에서** 할 수 있게 해주는 통합 개발 환경.

| IDE | 주요 언어 | 특징 |
|---|---|---|
| **VS Code** | 거의 모든 언어 | 가장 보편적, 확장 기능 풍부 |
| **Cursor** | 거의 모든 언어 | VS Code 기반 + AI 코딩 내장 |
| **PyCharm** | Python | Python 전용 기능 강력 |
| **IntelliJ** | Java/Kotlin | JetBrains 대표 IDE |
