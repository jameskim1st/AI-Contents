# 프로토콜과 HTTP

**Category:** 네트워크 / 기초
**Status:** stable
**Last updated:** 2026-04-12

## TL;DR

프로토콜(Protocol)은 컴퓨터 간 네트워크 통신을 위한 규격(규칙)이다. 가장 널리 쓰이는 프로토콜은 **HTTP**로, 브라우저에서 웹 정보를 송수신할 때 사용된다. 보안이 강화된 **HTTPS**가 현재 대부분의 웹사이트 표준이다.

## 설명

### 프로토콜이란

네트워크는 특정 개인이나 단체가 소유하는 개념이 아니다. 전 세계적으로 네트워크 통신이 가능한 이유는 **공통된 규격(규칙)** 이 존재하기 때문이며, 이를 **통신 프로토콜(Communication Protocol)** 이라 한다.

네트워크 통신을 하려면 "**어떤 방식(프로토콜)** 으로, **어디 컴퓨터(IP주소)** 에, **어떤 프로그램(PORT)** 을 연결할 것인지"를 정해야 한다.

### HTTP / HTTPS

- **HTTP (HyperText Transfer Protocol):** 브라우저에서 페이지 로드, 파일 다운로드 등 정보를 받아올 때 사용하는 프로토콜. 네트워크 통신에서 가장 많이 쓰인다.
- **HTTPS (HTTP + Secure):** HTTP에 보안(S)을 강화한 프로토콜. 통신 과정에서 데이터를 암호화한다. 대부분의 웹사이트가 HTTP를 막고 HTTPS를 사용하도록 권장한다.

브라우저 주소창에 `google.com`을 입력하면 자동으로 `https://google.com`이 적용되어 HTTPS 프로토콜로 통신한다.

### 주요 프로토콜 종류

| 프로토콜 | 용도 | 기본 PORT |
|---|---|---|
| HTTP | 웹 정보 송수신 | 80 |
| HTTPS | 보안 웹 통신 | 443 |
| SSH | 원격 컴퓨터 접속 | 22 |
| SMTP | 이메일 전송 | 25 / 587 |
| FTP | 파일 전송 | 21 |

### OSI 7계층 (심화)

컴퓨터 OS가 네트워크 신호를 특정 프로그램으로 전달하는 과정은 크게 **7개의 계층(Layer)** 으로 나뉜다. 이를 **OSI 7계층**이라 하며, 각 계층마다 역할이 정해져 있다. 네트워크 전공 도서에서 필수적으로 다루는 개념이다.

### 바이브코딩과의 연결

바이브코딩 도구(Bolt.new 등)가 자동으로 서버를 세팅해 줄 때 HTTPS 프로토콜이 기본 적용된다. API 호출 시 요청 URL 앞에 붙는 `https://`가 바로 이 프로토콜을 지정하는 부분이다.
