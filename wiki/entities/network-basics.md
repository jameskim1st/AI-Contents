# Network Basics (네트워크 기초)

**Category:** IT 기초 / 네트워크
**Status:** stable
**Last updated:** 2026-04-12

## TL;DR

네트워크는 **컴퓨터와 컴퓨터 간의 연결**이다. 연결이 성립하려면 **프로토콜**(어떤 방식으로), **IP 주소**(어디 컴퓨터에), **PORT**(어떤 프로그램을) 세 가지가 필요하다. 요청하는 쪽이 **클라이언트**, 응답하는 쪽이 **서버**이며, 서버는 특별한 슈퍼컴퓨터가 아니라 요청에 응답하도록 설정된 프로그램이 실행 중인 컴퓨터일 뿐이다.

## 설명

### 네트워크의 3대 요소

네트워크 통신이 이루어지려면 다음 세 가지를 알아야 한다:

| 요소 | 질문 | 예시 |
|---|---|---|
| **프로토콜 (Protocol)** | 어떤 방식으로 통신할 것인가? | HTTP, HTTPS, SSH, SMTP |
| **IP 주소** | 어느 컴퓨터에 연결할 것인가? | 192.168.0.1, 13.225.100.42 |
| **PORT** | 그 컴퓨터의 어떤 프로그램에 접근할 것인가? | 80(HTTP), 443(HTTPS), 3000(개발 서버) |

> 쉽게 말해: "**HTTPS** 방식으로, **13.225.100.42** 컴퓨터의, **443번** 포트에서 돌아가는 프로그램에 연결해줘."

### 클라이언트와 서버 — 식당 비유

Server의 어원은 "Serve하는 자". 식당에 비유하면 이해가 쉽다:

| 역할 | 네트워크 | 식당 비유 |
|---|---|---|
| **클라이언트 (Client)** | 정보를 **요청하는 쪽** — 브라우저, 앱 | **손님** — "파스타 하나 주세요" |
| **서버 (Server)** | 정보를 **제공하는 쪽** — 요청을 받아 처리 | **주방** — 주문을 받아 요리를 만듦 |
| **서비스 (Service)** | 서버가 **제공하는 것** — 웹페이지, 날씨 정보, AI 답변 | **음식** — 주방에서 나온 결과물 |
| **API** | 클라이언트와 서버 사이의 **표준화된 창구** | **종업원** — 주문을 전달하고 음식을 서빙 |

고객은 주방에 직접 들어가지 않듯, 프론트엔드도 백엔드 서버에 직접 접근하지 않고 반드시 **API(종업원)**를 통해 데이터를 요청하고 받는다.

### 서버의 종류

서버는 **목적에 맞는 프로그램이 실행 중인 컴퓨터**다:

- **웹 서버** — HTML/CSS/JS 파일을 전달 (Apache, Nginx)
- **API 서버** — 클라이언트 요청에 데이터를 JSON으로 응답
- **이미지 서버** — 이미지 파일을 저장·전달 (CDN)
- **비디오 서버** — 동영상 스트리밍 제공

> **1개 컴퓨터에 여러 서버를 띄울 수 있다.** CPU·메모리 성능이 좋아지면서 하나의 물리 머신에 웹 서버 + API 서버 + DB 서버를 함께 운영하는 것도 흔하다. 각 서버가 서로 다른 PORT를 사용하면 된다.

### 프론트엔드 vs 백엔드

| 영역 | 위치 | 역할 | 기술 |
|---|---|---|---|
| **프론트엔드** | 클라이언트 쪽 | 사용자가 보는 화면 | HTML, CSS, JS, React |
| **백엔드** | 서버 쪽 | 데이터 저장·인증·계산 (눈에 안 보임) | Python, Node.js, DB |

### 네트워크 물리적 수단

- **광케이블** — 0과 1의 전기 신호를 **빛의 물리적 신호**로 변환하여 전송
- **와이파이** — **전자기파**를 이용한 무선 통신
- **랜카드 (NIC)** — 컴퓨터가 네트워크 장치(유선/무선)를 이용하기 위한 하드웨어

### 프로토콜 주요 종류

| 프로토콜 | 용도 | 비고 |
|---|---|---|
| **HTTP** | 웹 정보 전송 | 가장 많이 쓰이는 프로토콜 |
| **HTTPS** | HTTP + 보안(암호화) | 대부분의 웹사이트 표준 |
| **SSH** | 원격 컴퓨터 접속 | 서버 관리에 필수 |
| **SMTP** | 이메일 전송 | 메일 발송 프로토콜 |
