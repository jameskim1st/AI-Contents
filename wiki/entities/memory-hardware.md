# Memory (하드웨어)

**Category:** IT 기초 / 하드웨어
**Status:** stable
**Last updated:** 2026-04-12

## TL;DR

컴퓨터 메모리(RAM)는 전기적 신호를 **임시로 저장**하는 하드웨어다. CPU의 계산 결과를 보관하고, 프로그램이 실행되는 공간을 제공한다. 프로그램을 종료하면 메모리에서 사라지는 **휘발성** 저장소이며, 영구 저장은 디스크(SSD/HDD)가 담당한다. 프로그래밍 작업이라면 **16GB 이상**을 권장한다.

> **NOTE:** 이 entity는 **하드웨어 RAM**에 관한 것이다. AI 에이전트의 상태 유지 메커니즘(단기·장기 메모리)은 [Memory (Agent)](./memory.md)를 참조.

## 설명

### 메모리의 2대 역할

1. **CPU 계산 결과 저장** — CPU가 연산하면서 나오는 중간·최종 결과를 임시 보관한다. CPU는 이를 다시 읽어 후속 계산에 사용한다.
2. **프로그램 실행 공간 제공** — 프로그램이 실행되면 **프로세스** 형태로 메모리 위에 올라간다. 프로그램이 종료되면 메모리에서 사라진다.

### RAM vs ROM

| 구분 | RAM (Random Access Memory) | ROM (Read Only Memory) |
|---|---|---|
| **휘발성** | O — 전원 꺼지면 데이터 사라짐 | X — 전원 꺼져도 유지 |
| **용도** | 프로그램 실행 공간, CPU 작업 버퍼 | BIOS 등 펌웨어 저장 |
| **속도** | 매우 빠름 (나노초 단위) | 빠르지만 RAM보다 느림 |
| **용량** | 8~128GB (사용자가 증설 가능) | 수 MB (고정) |

### 메모리 vs 디스크 — 핵심 구분

| 항목 | 메모리 (RAM) | 디스크 (SSD/HDD) |
|---|---|---|
| **저장 기간** | 임시 (전원 끄면 소멸) | 영구 (전원 꺼도 유지) |
| **속도** | 매우 빠름 | 상대적으로 느림 |
| **역할 비유** | 작업 **책상** — 지금 펼쳐놓은 서류 | **서랍장** — 보관해둔 파일 |
| **OS 관점** | 프로세스를 올리는 공간 | 프로그램을 설치·저장하는 공간 |

메모리가 프로그램 **실행 과정**을 책임진다면, 디스크는 프로그램을 컴퓨터에 **저장하는 과정**을 책임진다.

### 용량별 체감

- **8GB** — 웹 브라우징, 문서 작업은 가능하지만 탭 많이 열면 느려짐
- **16GB** — 프로그래밍, 가벼운 AI 작업에 적합 (최소 권장)
- **32GB+** — 대규모 데이터 분석, 로컬 LLM 실행, 영상 편집 등
- **64GB~** — 서버·워크스테이션급. 로컬에서 7B 이상 모델 구동 시 필요

### 프로세스와 메모리 구조

프로그램이 실행되어 프로세스가 되면, 메모리 안에 **Stack**(함수 호출 정보, 지역 변수)과 **Heap**(동적 할당 데이터) 영역이 생긴다. 자세한 내용은 [Program & Process](./program-process.md) 참조.
