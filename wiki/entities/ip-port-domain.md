# IP, PORT, 도메인

**Category:** 네트워크 / 기초
**Status:** stable
**Last updated:** 2026-04-12

## TL;DR

네트워크 통신에는 세 가지 주소 정보가 필요하다. **IP**는 컴퓨터의 네트워크 주소, **PORT**는 컴퓨터 내 특정 프로세스의 주소, **도메인(Domain Name)** 은 IP를 사람이 읽을 수 있도록 만든 영문 주소다. 방화벽은 IP와 PORT를 기반으로 연결을 제한한다.

## 설명

### IP (Internet Protocol Address)

IP는 **네트워크 통신을 위한 주소**다. 컴퓨터의 고유 주소가 아니라, 연결된 네트워크(와이파이, 랜선 등)를 기준으로 부여받는다.

- **IPv4:** 주소 개수가 255^4(약 43억 개)만큼 가능. 전통적인 IP 형식 (예: `192.168.0.1`).
- **IPv6:** IP 주소 포화 문제를 해결하기 위해 등장. 유심칩을 사용하는 스마트폰 IP가 대표적인 IPv6 사례.

IT 회사에서는 네트워크 공유기로 미리 IP 주소를 할당받아 회사 컴퓨터에 배정한다. 이렇게 하면 보안 유지와 관리가 수월하다.

### PORT

PORT는 **컴퓨터 내에서 프로세스에 도달하기 위한 주소**다. IP가 "어떤 컴퓨터인지"를 가리킨다면, PORT는 "그 컴퓨터 안의 어떤 프로그램인지"를 가리킨다.

| 프로토콜 | 기본 PORT |
|---|---|
| HTTP | 80 |
| HTTPS | 443 |

네트워크 주소에 IP만 적으면 해당 프로토콜의 기본 PORT가 자동 적용된다. 예를 들어 브라우저에서 네이버에 접속하면 네이버 웹서버가 입력된 PORT 번호를 이용해 웹 전달 프로세스에 연결해준다.

### 도메인 (Domain Name)

도메인 네임은 **사람이 쉽게 인식할 수 있는 영문 주소**다.

```
사용자 입력: naver.com
       ↓ DNS(네임 서버) 조회
실제 통신:  223.130.195.200 (IP)
```

브라우저에서 입력하는 영문 주소는 **DNS(Domain Name System, 네임 서버)** 에 등록된 주소다. 도메인을 사용하면 DNS를 거쳐 IP를 반환받는 과정이 자동으로 이루어진다.

서버에 도메인을 붙이려면 호스팅 서비스(가비아, Cloudflare 등)에서 도메인을 구매하고 DNS에 등록해야 한다.

### 방화벽 (Firewall)

네트워크 보안을 위해 컴퓨터 간 연결을 제한하는 시스템이다. **IP 주소**와 **PORT 번호** 등 네트워크 정보를 기반으로 허용/차단 규칙을 설정한다. 서버 운영 시 필요한 PORT만 열어두는 것이 기본 보안 원칙이다.
